# Plan: Reorganize Jekyll directories into `jekyll/` subfolder

## Goal
Move `_includes/`, `_layouts/`, and `_plugins/` into a `jekyll/` subdirectory to clean up the project root.

## Step 1: Remove unused `jekyll-paginate-content` plugin
The plugin is configured with `auto: false` and uses `--page-break--` as the separator, but **no posts contain `--page-break--`**. The `paginate: true` front matter on all 25 posts has no effect. Remove it.

1. Remove `gem "jekyll-paginate-content"` from `Gemfile`
2. Remove `jekyll-paginate-content` from the `plugins:` list in `_config.yml`
3. Remove the entire `paginate_content:` block from `_config.yml`
4. Remove `paginate: true` from the front matter of all 25 posts in `_posts/`
5. Run `bundle install` to update the lockfile

## Step 2: Upgrade Jekyll 3.9.3 → 4.x
The `plugins_dir` config option (needed to relocate `_plugins/`) is only available in Jekyll 4.0+.

1. Update `Gemfile`: change `gem "jekyll"` to `gem "jekyll", "~> 4.4"`
2. Run `bundle update jekyll` to update the lockfile and all dependencies
3. Run `bundle exec jekyll build` to verify the build succeeds

### Breaking changes to verify
- `post_url` tag now auto-prepends `baseurl` — search for `{{ site.baseurl }}/{% post_url` patterns (grep found none)
- Default `exclude` list is now **merged** (not overridden) with the user's `exclude` config — no action needed
- Template caching changes — should not affect this project since custom plugins are simple tag/filter generators

## Step 3: Move directories
1. `mkdir -p jekyll/_includes jekyll/_layouts jekyll/_plugins`
2. `git mv _includes/* jekyll/_includes/`
3. `git mv _layouts/* jekyll/_layouts/`
4. `git mv _plugins/* jekyll/_plugins/`
5. `rmdir _includes _layouts _plugins`

## Step 4: Update `_config.yml`
Add three configuration keys:
```yaml
layouts_dir: jekyll/_layouts
includes_dir: jekyll/_includes
plugins_dir: jekyll/_plugins
```

No other file changes needed — `{% include %}` tags and `layout:` front matter resolve via these config values automatically.

## Step 5: Verify
```
bundle exec jekyll build
```

## What does NOT need changing
- All 52 `{% include %}` references across layouts/includes — resolved via `includes_dir`
- All 52 `layout:` references across pages/posts — resolved via `layouts_dir`

## Optional: move `_sass/` too
The `sass_dir` config option has been available since Jekyll 2.x. `_sass/` (28 SCSS files) could also move to `jekyll/_sass/` by adding `sass_dir: jekyll/_sass` to `_config.yml`.
