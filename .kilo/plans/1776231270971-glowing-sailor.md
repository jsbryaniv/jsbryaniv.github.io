# Plan: Live-Coding Animation for Quarto Tutorials

## Problem Summary

You want code blocks in your tutorials to animate like live coding when scrolled into view:
1. Code types out character by character with a blinking cursor
2. Output appears line by line after code finishes

**Two issues prevent this from working:**

### Issue 1: CSS/JS Not Loaded in Rendered HTML

The `_quarto.yml` format settings are NOT applied to files under `_tutorials/` because:
- Directories with `_` prefix are excluded from Quarto project scope (by design)
- The document's `format: gfm: default` in front matter doesn't include HTML settings
- When rendering individual QMD files outside the project, `_quarto.yml` format options don't apply

### Issue 2: JavaScript Destroys Syntax Highlighting

The current `code-animation.js` does:
```javascript
code.innerText = "";      // Destroys all <span> elements
code.innerText += line;   // Writes plain text without colors
```

But Quarto generates syntax-highlighted code as nested `<span>` elements:
```html
<code class="sourceCode">
  <span id="cb1-1"><span class="kw">def</span> foo():</span>
  <span id="cb1-2">    <span class="cf">return</span> x</span>
</code>
```

The JS needs to preserve these spans to maintain syntax coloring.

---

## Solution

### Part 1: Ensure CSS/JS Loads in Tutorial HTML

**Option A (Recommended): Add format block to tutorial front matter**

Since tutorials render individually outside the project, add the HTML format config directly to each tutorial's YAML:

```yaml
format:
  html:
    toc: true
    code-copy: true
    css: /assets/code-animation/code-animation.css
    include-after-body:
      text: |
        <script src="/assets/code-animation/code-animation.js"></script>
  gfm: default
```

**Option B: Use metadata-files for shared config**

Create a shared metadata file and reference it:

1. Create `assets/code-animation/_tutorial-format.yml`:
   ```yaml
   format:
     html:
       toc: true
       code-copy: true
       css: /assets/code-animation/code-animation.css
       include-after-body:
         text: |
           <script src="/assets/code-animation/code-animation.js"></script>
   ```

2. In each tutorial, add:
   ```yaml
   metadata-files:
     - /assets/code-animation/_tutorial-format.yml
   ```

**Recommendation**: Option A is simpler and more explicit. The extra lines in front matter are worth the clarity.

### Part 2: Rewrite JavaScript for Quarto's HTML Structure

Rewrite `code-animation.js` to:

1. **Preserve syntax highlighting**: Work with existing `<span>` elements instead of destroying them
2. **Type character by character**: Walk text nodes inside spans, revealing one character at a time
3. **Trigger on scroll**: Use `IntersectionObserver` instead of running on page load
4. **Animate output separately**: Reveal output lines one at a time after code finishes

**New JS approach:**
- Find all `<span id="cb1-N">` line spans inside `<code class="sourceCode">`
- Hide all line spans initially (`visibility: hidden`)
- When block scrolls into view, reveal spans one at a time
- Within each span, type out text nodes character by character (preserving child spans)
- After code finishes, reveal output block line by line
- Show blinking cursor during typing, hide when done

### Part 3: Update CSS to Match New Approach

The current CSS uses `opacity: 0` on `pre code` which is fine for initial hide, but:
- The cursor `::after` pseudo-element should only show during typing
- Need class hooks for JS to toggle typing/done states
